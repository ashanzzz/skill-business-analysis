## 二、四轮验证框架（必须执行）

### 为什么需要四轮？

```
单一来源 → 容易被误导
两源交叉 → 可能一致但不能保证
三源验证 → 更接近真实
四轮推理 → 更可靠
```

### 四轮验证操作流程

```
┌────────────────────────────────────────────────────────┐
│                    四轮验证流程（必须执行）                        │
├────────────────────────────────────────────────────────┤
│                                                             │
│ 第一轮：多源数据验证                                         │
│ ─────────────────────────────────────────────────        │
│ 工具：web_search（并行）                                     │
│ 方法：                                                       │
│   1. 搜索3个独立来源                                       │
│      - 来源A：官方数据（World Bank/Gartner/政府）            │
│      - 来源B：研究报告（McKinsey/BCG/协会）               │
│      - 来源C：公开数据（财报/新闻/平台）                     │
│   2. 并行获取页面内容                                       │
│   3. 提取具体数字+来源+日期                                 │
│                                                             │
│ 输出：                                                       │
│   来源A：[数字] [来源] [日期]                              │
│   来源B：[数字] [来源] [日期]                              │
│   来源C：[数字] [来源] [日期]                              │
│   一致性：✅一致 / ⚠️有分歧（±X%）                         │
│                                                             │
│ 第二轮：逻辑自洽验证                                         │
│ ─────────────────────────────────────────────────        │
│ 工具：AI推理（基于常识+规则）                                 │
│ 方法：                                                       │
│   1. 供给=需求？（长期均衡）                                │
│   2. Σ市场份额≤100%？                                     │
│   3. LTV>3×CAC？（如适用）                                │
│   4. 增速<需求增速？（否则不可持续）                         │
│   5. 价格>边际成本？（可持续）                              │
│   6. TAM>SAM>SOM？（逻辑递减）                             │
│                                                             │
│ 输出：                                                       │
│   逻辑1：✅成立 / ❌不成立 [原因]                          │
│   逻辑2：✅成立 / ❌不成立 [原因]                          │
│   ...                                                       │
│                                                             │
│ 第三轮：案例类比验证                                         │
│ ─────────────────────────────────────────────────        │
│ 工具：web_search（找同类案例）                               │
│ 方法：                                                       │
│   1. 搜索同类市场的已知案例                                 │
│   2. 对比案例与目标市场的相似性                             │
│   3. 类比推理验证结论                                       │
│                                                             │
│ 输出：                                                       │
│   案例：[名称] [类比结论] [相似度：高中低]                   │
│   适用性：✅适用 / ⚠️有限适用 / ❌不适用                     │
│                                                             │
│ 第四轮：时间序列验证                                         │
│ ─────────────────────────────────────────────────        │
│ 工具：web_search（历史数据）                                 │
│ 方法：                                                       │
│   1. 获取历史数据（3-5年）                                 │
│   2. 计算历史CAGR                                          │
│   3. 对比当前预测与历史趋势                                  │
│                                                             │
│ 输出：                                                       │
│   历史增速：[X%]                                            │
│   预测增速：[Y%]                                            │
│   一致性：✅合理 / ⚠️偏离 / ❌矛盾                           │
│                                                             │
├────────────────────────────────────────────────────────┤
│ 验证结果判定                                                 │
│ ─────────────────────────────────────────────────        │
│ 四轮全部一致 → 🟢 极高可信                                   │
│ 三轮一致 → 🟢 高可信                                        │
│ 两轮一致 → 🟡 中可信                                        │
│ 两轮分歧 → 🟠 低可信，标注分歧                              │
│ 四轮全部分歧 → 🔴 不采用，需重新收集数据                     │
└────────────────────────────────────────────────────────┘
```

## 三、十二维分析框架

### D1：市场规模分析

**分析内容：**
- 全球TAM/SAM/SOM
- 区域分解
- 历史趋势
- 未来预测

**分析方法：**

| 方法 | 公式 | 数据来源 |
|------|------|---------|
| 自上而下 | TAM = GDP × 行业占比 | World Bank + 行业协会 |
| 自下而上 | TAM = Σ(细分) = 用户数 × 客单价 × 频次 | 平台数据 + 调研 |
| 供给端加总 | TAM = Σ(玩家收入) ÷ 集中度 | 财报 + 行业协会 |
| 需求端推算 | TAM = 目标用户 × 渗透率 × 客单价 | 人口 + 消费数据 |
| 历史外推 | TAM(n) = TAM(0) × (1+CAGR)^n | 历史数据 |

**输出格式：**
```markdown
## D1：市场规模

### 全球市场规模

| 加权平均 | **$XX亿** | - | 🟢高 |
```

### D10：财务建模分析

**核心计算公式：**

```
TAM = 用户数 × 客单价 × 年均频次
SAM = TAM × 可服务比例
SOM = SAM × 可获得比例

LTV = 用户年均收入 × 毛利率 × 留存年限
     = ARPU × 毛利率 × (1/(1-留存率))

CAC = 总获客成本 ÷ 新增客户数

LTV/CAC = LTV ÷ CAC（>3x为健康）

CAC回收期 = CAC ÷ (ARPU × 毛利率)（<12月为健康）

YoY增长率 = (当年 - 上年) ÷ 上年 × 100%

CAGR = (末年/首年)^(1/年数) - 1
```

**单位经济标准：**

| 指标 | 健康 | 可接受 | 危险 |
|------|------|--------|------|
| LTV/CAC | >3x | 1-3x | <1x |
| CAC回收期 | <12月 | 12-18月 | >18月 |
| 毛利率 | >60% | 40-60% | <40% |

**行业估值倍数：**

| 行业 | EV/Revenue | EV/EBITDA | P/E |
|------|-----------|-----------|-----|
| 软件/SaaS | 5-20x | 20-40x | 20-40x |
| 电商 | 0.5-2x | 10-20x | 15-30x |
| 金融 | 2-5x | 8-15x | 10-20x |
| 制造 | 1-3x | 8-15x | 12-25x |

### D12：投资决策分析

**决策框架：**

| 阶段 | 决策标准 | 通过条件 |
|------|---------|---------|
| 机会筛选 | TAM > $10亿 | ✅ TAM达标 |
| 定量筛选 | NPV > 0 | ✅ NPV>0 |
| 定量筛选 | IRR > WACC+8% | ✅ IRR>15% |
| 定量筛选 | 回本周期 < 3年 | ✅ <3年 |
| 定性筛选 | 团队能力 | ✅ 强 |
| 定性筛选 | 护城河 | ✅ 可持续 |
| 最终决策 | 风险收益比 | ✅ R/R<2 |

**NPV计算：**
```
NPV = Σ(CFt / (1+r)^t) - I
其中：CFt = 第t年现金流；r = 折现率；I = 初始投资
```

**IRR计算：**
```
NPV = 0 时的折现率r
可用Excel/计算器：=IRR(CF0:CFn)
```

**风险收益评估矩阵：**

| 风险/收益 | 高收益 | 中收益 | 低收益 |
|----------|--------|--------|--------|
| 高风险 | 观望 | 不投 | 不投 |
| 中风险 | 积极参与 | 谨慎 | 不投 |
| 低风险 | **强烈推荐** | 观察 | 保守 |
